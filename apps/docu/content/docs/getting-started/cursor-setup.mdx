Complete guide to configuring Cursor for optimal AI-assisted development.

## Prerequisites

- Cursor IDE installed
- This repository cloned

## Step 1: Add Documentation Sites

Configure Cursor to access documentation:

1. Open Cursor Settings (Cmd/Ctrl + ,)
2. Navigate to "Features" → "Docs"
3. Add the following documentation sites:
   - [Vencura Docs](https://vencura-docs.vercel.app/docs) (`https://vencura-docs.vercel.app/docs`)
   - [Drizzle ORM Docs](https://orm.drizzle.team/docs) (`https://orm.drizzle.team/docs`)
   - [TanStack Query Docs](https://tanstack.com/query/latest/docs) (`https://tanstack.com/query/latest/docs`)
   - [Fastify Docs](https://fastify.dev/docs) (`https://fastify.dev/docs`)
   - [Fumadocs](https://fumadocs.dev/docs) (`https://fumadocs.dev/docs`)

These documentation sites help Cursor provide better context-aware assistance when working with these technologies. Click the links to visit, or copy the URLs in backticks to add to Cursor settings.

These indexed documentation sources are crawled and embedded to improve AI accuracy. For details on how Cursor Docs indexing works and how the AI uses these sources, see [AI Workflow - Cursor Docs Indexing](/docs/getting-started/ai-workflow#cursor-docs-indexing).

## Step 2: Configure MCP Servers

### What are MCP Servers?

Model Context Protocol servers extend AI capabilities with specialized tools. They allow Cursor to interact with external services and perform actions beyond code editing.

### Available MCP Servers

This project includes 7 MCP servers:

1. **shadcnui-official** - Canonical shadcn/ui component definitions and variants
2. **shadcnui-jpisnice-react** - shadcn/ui v4 blocks, demos, and page templates for React
3. **shadcnui-jpisnice-react-native** - shadcn/ui v4 components, blocks, and demos for React Native
4. **github** - Repository operations and issue management
5. **coderabbit** - CodeRabbit AI code review comments and PR insights
6. **vencura-docu** - Vencura project documentation context
7. **vencura-fastify** - Vencura API package context

**Note:** `shadcnui-jpisnice-react-native` is configured for future React Native support.

### Setting Up API Keys

MCP servers require API keys to function:

**For zsh (recommended):**
```bash
# Add to ~/.zshrc
export GITHUB_TOKEN=your_github_token_here  # Required for github and shadcnui-jpisnice-react

# Reload
source ~/.zshrc
```

**For bash:**
```bash
# Add to ~/.bashrc
export GITHUB_TOKEN=your_github_token_here  # Required for github and shadcnui-jpisnice-react

# Reload
source ~/.bashrc
```

**Important:** Restart Cursor after setting environment variables.

### Getting API Keys

- **GITHUB_TOKEN**: [GitHub Personal Access Tokens](https://github.com/settings/tokens) (required for GitHub MCP, CodeRabbit MCP, and shadcnui-jpisnice-react)

### Verifying MCP Setup

The MCP configuration is already in `.cursor/mcp.json`. After setting API keys:

1. Restart Cursor
2. Open Cursor settings
3. Navigate to "MCP Servers"
4. Verify all servers show as "Connected"

## Troubleshooting

### MCP Servers Not Connecting

- Verify API keys are exported in shell
- Restart Cursor completely
- Check `.cursor/mcp.json` syntax
- Ensure environment variables are loaded in shell

### CodeRabbit MCP Not Working

If the CodeRabbit MCP server fails to connect:

1. **Verify package name**: The correct package is `coderabbitai-mcp@latest` (not `@coderabbitai/mcp-server`)
2. **Check environment variable**: CodeRabbit MCP expects `GITHUB_PAT` but reads from your global `GITHUB_TOKEN` (configured in `.cursor/mcp.json`)
3. **Test manually**: Run `GITHUB_PAT="$GITHUB_TOKEN" pnpm dlx coderabbitai-mcp@latest --help` to verify the package works
4. **Verify token scopes**: Ensure your GitHub token has `repo` scope for private repositories or `public_repo` for public repositories
5. **Restart Cursor**: After fixing configuration, fully restart Cursor to reload MCP servers

**Common issues:**
- ❌ Wrong package: `@coderabbitai/mcp-server` (doesn't exist)
- ✅ Correct package: `coderabbitai-mcp@latest`
- ❌ Wrong env var: `GITHUB_TOKEN` directly
- ✅ Correct mapping: `GITHUB_PAT: "${GITHUB_TOKEN}"` (maps your token to the expected variable name)

### Cursor Rules Not Loading

- Check `.cursor/rules/` directory exists
- Verify rule files have `.mdc` extension
- Restart Cursor
- Check Cursor settings for rules configuration

### Documentation Sites Not Working

- Verify URLs are correct
- Check internet connection
- Restart Cursor
- Check Cursor settings → Features → Docs
